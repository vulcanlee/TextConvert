# TextConvert 使用者操作說明

- 文件用途:說明如何使用發布版 TextConvert 執行檔進行文件批次轉換。
- 主要讀者:一般使用者。
- 文件版本:1.0
- 對應系統版本:1.0.1
- 建立日期:2026-07-15
- 編碼:UTF-8 繁體中文

---

## 一、這個工具是什麼

TextConvert 會把您指定資料夾內的多種文件(Word、PowerPoint、Excel、PDF、Markdown、
純文字、HTML、CSV、JSON、XML),批次擷取成**純文字檔(.txt)**,存到另一個您指定的資料夾,
方便後續作為知識庫、搜尋或 AI 處理的素材。資料夾內不支援的檔案會自動略過。

## 二、系統需求

- Windows 64 位元作業系統。
- **不需要另外安裝 .NET 執行環境**(執行檔已內含所需元件)。

## 三、您會收到的檔案

| 檔案 | 說明 | 是否必要 |
|------|------|:--------:|
| `TextConvert.exe` | 主程式 | 必要 |
| `appsettings.json` | 設定檔(來源/輸出目錄、日誌等) | 必要 |
| `使用者操作說明.md` | 本說明文件 | 參考 |
| `TextConvert.pdb` | 除錯符號檔,一般使用可忽略 | 否 |

> 請將以上檔案放在**同一個資料夾**內。程式執行時會讀取旁邊的 `appsettings.json`。

## 四、快速開始

1. 開啟「命令提示字元(Command Prompt)」或「PowerShell」。
2. 切換到 `TextConvert.exe` 所在的資料夾,例如:
   ```
   cd C:\TextConvert
   ```
3. 執行轉換(擇一方式,見下一節)。
4. 完成後到您指定的輸出資料夾查看產生的 `.txt` 檔。

## 五、兩種指定「來源 / 輸出資料夾」的方式

### 方式 A:用命令列參數(適合臨時指定)

```
TextConvert.exe --source "D:\我的文件" --output "D:\轉換結果"
```

### 方式 B:寫在 appsettings.json(適合固定使用)

用記事本開啟 `appsettings.json`,填入來源與輸出資料夾後存檔,之後直接執行:

```json
{
  "Version": "1.0.1",
  "Conversion": {
    "SourceDirectory": "D:\\我的文件",
    "OutputDirectory": "D:\\轉換結果",
    "Recursive": true,
    "Overwrite": true
  }
}
```

然後執行:

```
TextConvert.exe
```

> 注意:JSON 內的路徑反斜線要寫成兩個(`\\`)。若命令列有給 `--source` / `--output`,
> 會**優先覆寫**設定檔中的對應設定。

## 六、appsettings.json 設定說明

| 設定 | 說明 |
|------|------|
| `Conversion.SourceDirectory` | 來源資料夾(要轉換的檔案放這裡) |
| `Conversion.OutputDirectory` | 輸出資料夾(轉出的 `.txt` 存這裡,不存在會自動建立) |
| `Conversion.Recursive` | `true` 會連同子資料夾一起處理;`false` 只處理最上層 |
| `Conversion.Overwrite` | `true` 會覆寫已存在的輸出檔;`false` 則略過已存在者 |
| `Serilog.MinimumLevel.Default` | 日誌詳細程度(`Information` / `Warning` / `Error` 等) |

## 七、命令列參數

| 參數 | 縮寫 | 說明 |
|------|------|------|
| `--source <目錄>` | `-s` | 指定來源資料夾 |
| `--output <目錄>` | `-o` | 指定輸出資料夾 |
| `--help` | `-h`、`-?` | 顯示說明後結束 |
| `--version` | `-v` | 顯示版本後結束 |

範例:

```
TextConvert.exe -h
TextConvert.exe --version
TextConvert.exe -s "D:\我的文件" -o "D:\轉換結果"
```

## 八、支援格式

| 類別 | 副檔名 |
|------|--------|
| Word | `.docx` |
| Excel / 表格 | `.xlsx`、`.csv` |
| PowerPoint | `.pptx` |
| PDF | `.pdf`(僅擷取內嵌文字) |
| 標記 / 網頁 | `.md`、`.markdown`、`.html`、`.htm` |
| 純文字 | `.txt`、`.text`、`.json`、`.xml` |

不在清單內的檔案(例如 `.zip`、舊版 `.doc`/`.xls`/`.ppt`、圖片)會被**自動略過**。

## 九、輸出結果規則

- 輸出資料夾會**鏡像**來源資料夾的子目錄結構。
- 每個檔案對應一個純文字檔,**保留原副檔名**再加上 `.txt`,例如:
  - `報告.docx` → `報告.docx.txt`
  - `資料.pdf` → `資料.pdf.txt`
- 這樣同名不同類型的檔案(如 `報告.docx` 與 `報告.pdf`)不會互相覆蓋。

## 十、日誌(執行紀錄)

- 執行過程會顯示在畫面上,同時寫入執行資料夾下的 `logs\textconvert-YYYYMMDD.log`。
- 日誌等級:
  - **Information**:一般流程與每個檔案的轉換結果。
  - **Warning**:例如檔案沒有可擷取的文字(可能是掃描影像型 PDF)。
  - **Error**:某個檔案轉換失敗(程式會**繼續處理其他檔案**,不會中斷)。

## 十一、常見情況與排除

| 情況 / 訊息 | 原因與處理 |
|------------|-----------|
| 「尚未指定來源或輸出目錄」 | 未在 `appsettings.json` 設定,也沒給 `--source`/`--output`。請補上其一。 |
| 「來源目錄不存在」 | 來源路徑打錯或資料夾不存在。請確認路徑。 |
| 某檔案「未擷取到任何文字」的警告 | 可能是掃描影像型 PDF(無內嵌文字);本工具不做 OCR,會輸出空內容。 |
| 某檔案轉換失敗(Error) | 該檔可能損毀或格式異常;程式會略過它並繼續處理其餘檔案,詳情見日誌。 |
| 執行後沒有畫面停留 | 這是命令列工具,請在命令提示字元/PowerShell 中執行以看到訊息與結果。 |

## 十二、完整操作範例

```
cd C:\TextConvert
TextConvert.exe --source "D:\客戶文件" --output "D:\知識庫素材"
```

畫面會依序顯示每個檔案的轉換情形,最後顯示總結,例如:

```
[INF] 轉換完成。成功=10, 跳過=1, 失敗=0, 總計=11
```

接著到 `D:\知識庫素材` 即可取得所有轉出的 `.txt` 檔。
